# docs/ について

このディレクトリには、git管理を始める前に行っていた試行錯誤に加えて、
`tprolog.pl`(型システム本体)が節目を迎えるたびに、その時点の実装を
まるごとコピーしたスナップショットを追加していきます。
「もう編集しない過去の遺物」ではなく、**現在進行系の型システムの歴史を
たどれるようにするための記録**という位置づけです。

- 実際の開発・利用は、リポジトリルートの `tprolog.pl` と、その利用例
  (`examples/` 以下、COPL演習群を集めた `examples/copl/` を含む) に対して行い、
  git のコミット/タグで細かい差分を追っていきます。
- 節目となる変更(型システムの仕組み自体が変わるような更新)があった
  ときに、その時点の `tprolog.pl` を `docs/tprolog_unionN.pl` として
  コピーし、このファイルの説明を合わせて更新します。
- このディレクトリのファイル自体は後から書き換えず、あくまで
  「その時点のスナップショット」として残します。

## おおまかな流れ

1. **`tprolog0.pl` → `tprolog1.pl` → `tprolog2.pl`**
   行多相(row polymorphism)を持つ簡易的な型システムの最初期の実験。
2. **`expand_bnf.pl` / `expand_bnf2.pl` / `expand_bnf copy.pl`**
   `::=` によるBNF風の型・述語シグネチャ宣言をterm_expansionで展開する
   仕組みの試作。
3. **`lambda.pl` → `lambda2.pl` → `lambda3.pl`**
   ラムダ計算・クロージャを持つ評価器の試作。
4. **`tprolog_union.pl` → `tprolog_union2.pl` 〜 `tprolog_union12.pl`**
   上記の要素を統合し、以下を段階的に導入していったメインの実験ライン。
   - `::=`宣言からの述語シグネチャ・型(カインド)の自動登録
   - カインドの宣言時整合性検証 (`check_kind_decl`)
   - 相互再帰する型のための遅延検証 (`check_all_kinds` によるまとめ検証)
   - 演算子オーバーロード時の誤判定の修正 (`+`/`*` など)
   - エイリアス判定の精緻化 (`is_true_alias_rhs`: 述語シグネチャ・
     既定義の型参照・`list/1`・単一構成子のnewtypeの区別)
5. **`tprolog_union13.pl`**
   `tprolog_union12.pl` の成果を汎用モジュール `tprolog.pl` として
   切り出した後、`tprolog.pl` に加えた変更を組み込んで、
   `union12` までと同じくテスト付きで単体で完結して動く
   (`swipl tprolog_union13.pl` だけで `run_tests` まで走る)ように
   保った版。モジュールのスナップショットを置くだけでなく、この形で
   残しておくと単体で挙動を再現・確認できる。
   - `type_check_all/0,1` の追加(カインド一括検証をまとめて呼ぶ
     ユーティリティ)
   - 節(clause)の型検査を `term_expansion` の中でその場(ロード時)に
     行うのをやめ、`pending_clause_check/3` に記録して
     `type_check_all/0,1` でまとめて検証するように変更
     (相互再帰する述語同士で、片方のシグネチャ登録前にもう片方が
     誤って弾かれる問題を、カインドの遅延検証と同じ発想で解消した)
   - 上記を確認する `deferred_clause_check` テストブロックを追加
     (プログラム中の節が全て型検査済みでエラーゼロであることの確認、
     わざと型の合わない節を `try_check_clause/4` で直接検証してみせる
     デモ、`type_check_all/1` の戻り値の形の確認)
   - `check_all_kinds/0`(カインドエラーだけを表示する版)を削除し、
     `type_check_all/0` に統合。`type_check_all/0` は元々
     `check_all_kinds/0` の存在を知らずに同じ目的で作られたもので、
     カインドエラー・節の型エラーの表示とサマリ表示まで含めた
     上位互換だったため、重複していた `check_all_kinds/0` 側を
     削除した(`check_all_kinds/1` は他から使われているのでそのまま残す)。
6. **`tprolog_union14.pl`**
   `tprolog_union13.pl` に、DCG規則(`-->`)への型付け対応を追加した版。
   - `term_expansion` に、DCG規則 `(Head --> Body)` 用の節を追加。
     非終端記号名にシグネチャが宣言されているものだけ、自前で
     `dcg_translate_rule/2` により変換し、変換後の通常節(差分リスト
     引数が2つ増える)を `pending_clause_check` に登録して型検査する。
     シグネチャの無いDCG規則は従来通りSWI標準のDCG変換に素通しする。
   - デモとして `digit_tok//1`(1文字の数字を整数に変換するDCG規則)を
     追加し、`dcg_clause_check` テストブロックで実際に動作すること・
     型検査されること・型の合わないDCG節が検出できることを確認した。
   - 実装過程で、DCG変換後の節が差分リストを繋ぐのに使う `(=)/2` の
     シグネチャを宣言し忘れて型エラーになる、という不具合を実際に
     踏んで直した(小さな独立した再現コードで切り分けて特定した)。
7. **`tprolog_union15.pl`**
   `tprolog_union14.pl` に、述語シグネチャ(`pred_sig`/`goal`)の
   オーバーロード対応を追加した版。
   - 以前の `pred_sig/2` は、同じ述語名に複数の引数リスト形式
     シグネチャが登録されていても、最初に見つかった1つに即座に
     カットしてコミットしてしまい、2つ目以降の候補を試せなかった
     (データ構成子側の `match_sig` は、`maplist` が失敗したら次の
     候補にバックトラックできるのに対して非対称だった)。
   - `pred_sig/2` を全候補を列挙可能な形にし、`goal/2` 側の
     `maplist(tp(Γ),Ms,Ts)` が成功するまでバックトラックしてから
     カットするように変更した(`match_sig` と同じ設計に統一)。
   - `examples/copl/EvalML5.pl` の `doesntMatch/2`(第2引数がスカラー
     値とリストの残りのどちらにもなりうる再帰述語)を型付けする際に
     必要になった。
   - デモとして `empty/1`・`shrink/1`(スカラー/リストの2つの
     シグネチャを持つ、意図的にオーバーロードした述語)を追加し、
     `list_and_overload` テストブロックで、両方のシグネチャで実際に
     動くこと・`pred_sig` が両方の候補を列挙できること・型の合わない
     使い方は引き続き検出されることを確認した。
   - 実際に `use_module(tprolog)` して同じ内容を検証するテストは
     `tests/list_tests.pl` に移植してある。

## 現在の到達点

`tprolog.pl` は `:- module(tprolog, [...])` として切り出したモジュールで、
`examples/sample.pl` / `examples/stlc.pl`(単純型付きラムダ計算) /
`examples/add.pl`(加算のみの最小言語) / `examples/let_poly.pl`
(let多相) / `examples/copl/` 以下のCOPL演習群を
`:- use_module(tprolog).` で利用する形になっています。
今後もこの構成のまま `tprolog.pl` に変更を加えつつ、節目ごとに
このディレクトリへスナップショットを追加していきます。
