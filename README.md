# tprolog

tprologはProlog 上に、`::=` という BNF 風の記法で型・カインド・述語シグネチャを宣言できる小さな型付けフレームワークです。
この型システムは、構造的な共用型(union型)として実装されているので、簡潔に記述することを可能にしています。
宣言した節はロード時に自動的に型検査され、意図しない型の値を渡す節はエラーとして弾かれます。

SWI-Prolog (動作確認: 10.0.2) が必要です。

## 使い方

```prolog
:- use_module(tprolog).

% 型 ty はいくつかの構成子を持つ直和型(カインド)
ty ::= tint | tbool | (ty->ty).

% 述語シグネチャの宣言
integer::= [_].
is     ::= [int_t, int_t].

% term は式の構文
term ::= int_t | term+term.

% typeof/2 などの節は、term/ty のシグネチャに基づいて
% ロード時に自動で型検査される
typeof ::= [term, ty].
typeof(I,tint):- integer(I).
typeof(E1+E2,tint):- typeof(E1,tint), typeof(E2,tint).

% 全ての ::= 宣言が出揃った後に、まとめてカインドの整合性を検証する
:- check_all_kinds(Results),
   ( member(error(_,_,_,_,_,_), Results) -> writeln(ng) ; writeln(ok) ).
```

実行例は `examples/add.pl`(加算のみの最小言語)、`examples/stlc.pl`(単純型付きラムダ計算)、`examples/sample.pl`(相互再帰する型・述語の例)を参照してください。

```sh
swipl examples/add.pl
swipl examples/stlc.pl
swipl examples/sample.pl
```

## 主な仕組み

- **`::=`**: BNF風に型(カインド)や述語のシグネチャを宣言します。
  - `T ::= A | B(T) | ...` のように `|` で複数の構成子を書くと、 `T` は自動的にカインドとして登録されます。
  - `T ::= [Args]` や `T ::= [Args]->Result` は述語シグネチャの宣言で、カインド登録はされません。
  - `T ::= X`(裸のアトム1つ)は、`X` が既に定義済みの型ならエイリアスとして、未定義の新しい記号なら単一構成子のカインドとして登録されます。
- **`check_all_kinds/0,1`**: 全ての `::=` 宣言が出揃った後に、カインドの整合性(構成子の引数が全てカインドであること)をまとめて検証します。
  相互再帰する型もこの時点で解決できます。
- **`check/2` (内部で `term_expansion` フック経由)**: `Head :- Body` や `Head` の形の節をロードする際、宣言済みのシグネチャに基づいて型検査し、型が合わない節はロード時にエラーとして弾きます。
- **`tp/3`**: 値が指定した型(カインド)に適合するかどうかを判定する、サブタイピングを考慮した中心的な型判定述語です。
- **`wf_kind/1`**: ある項が(いずれかの)カインドとして正しく整形されているかを判定します。

## 共用型(union型)としての型システム

この型システムは、OCamlのような明示的なタグ付きバリアント(タグ付き共用体)ではなく、構造的な共用型(union型)として実装されています。
`T ::= A | B | ...` で定義したカインドは「値がAでもBでも構わない」という集合的な条件であり、値そのものにはどの構成子由来かを示すタグは付きません。
そのため、整数や裸のアトムのようなタグの無い値であっても、それが属する複数のカインドに同時に適合できます。

```prolog
expr ::= int_t | atom_t | expr+expr.

?- tp([], 42, int_t).   % true (42 はタグなしの整数値)
?- tp([], 42, expr).    % true (int_t は expr の構成子の1つなので、同じ 42 がそのまま expr にも適合する)
?- tp([], foo, expr).   % true (裸のアトム foo も atom_t 経由で expr に適合する)
```

`42` を `int_t(42)` のように包み直す必要はなく、同じ値がそのまま複数の型に適合する点が、タグ付き代数的データ型との大きな違いです。

この型システムを既存の型理論の文脈に位置づけた解説と参考文献は
[docs/type-system.md](docs/type-system.md) にまとめてあります。
