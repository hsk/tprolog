# tprolog

Prolog 上に、`::=` という BNF 風の記法で型・カインド・述語シグネチャを
宣言できる小さな型付けフレームワークを実装したもの。宣言した節は
ロード時に自動的に型検査され、意図しない型の値を渡す節はエラーとして
弾かれる。

SWI-Prolog (動作確認: 10.0.2) が必要です。

## 使い方

```prolog
:- use_module(tprolog).

% 型 ty はいくつかの構成子を持つ直和型(カインド)
ty ::= tint | tbool | (ty->ty).

% 述語シグネチャの宣言
integer::= [_].
is     ::= [int_t, int_t].

% term は式の構文
term ::= int_t | term+term.

% typeof/2 などの節は、term/ty のシグネチャに基づいて
% ロード時に自動で型検査される
typeof ::= [term, ty].
typeof(I,tint):- integer(I).
typeof(E1+E2,tint):- typeof(E1,tint), typeof(E2,tint).

% 全ての ::= 宣言が出揃った後に、まとめてカインドの整合性を検証する
:- check_all_kinds(Results),
   ( member(error(_,_,_,_,_,_), Results) -> writeln(ng) ; writeln(ok) ).
```

実行例は `add.pl`(加算のみの最小言語)、`stlc.pl`(単純型付き
ラムダ計算)、`sample.pl`(相互再帰する型・述語の例)を参照してください。

```sh
swipl -q add.pl
swipl -q stlc.pl
swipl -q sample.pl
```

## ファイル構成

| ファイル | 内容 |
|---|---|
| `tprolog.pl` | 本体。サブタイピング・型検査・カインドシステムを提供するモジュール |
| `add.pl` | 加算だけの最小言語。`(1+2)+(3+4)` を型検査・評価するサンプル |
| `stlc.pl` | 単純型付きラムダ計算(STLC)。`typeof/3` と大ステップ評価器 `eval/3` |
| `sample.pl` | 相互再帰する型(`nat_even`/`nat_odd`)と述語(`even`/`odd`)の例 |
| `docs/` | git管理を始める前の試行錯誤の記録(参照専用、[docs/README.md](docs/README.md)参照) |

## 主な仕組み

- **`::=`**: BNF風に型(カインド)や述語のシグネチャを宣言する。
  - `T ::= A | B(T) | ...` のように `|` で複数の構成子を書くと、
    `T` は自動的にカインドとして登録される。
  - `T ::= [Args]` や `T ::= [Args]->Result` は述語シグネチャの宣言で、
    カインド登録はされない。
  - `T ::= X`(裸のアトム1つ)は、`X` が既に定義済みの型ならエイリアス、
    未定義の新しい記号なら単一構成子のカインドとして登録される。
- **`check_all_kinds/0,1`**: 全ての `::=` 宣言が出揃った後に、カインドの
  整合性(構成子の引数が全てカインドであること)をまとめて検証する。
  相互再帰する型もこの時点で解決できる。
- **`check/2` (内部で `term_expansion` フック経由)**: `Head :- Body` や
  `Head` の形の節をロードする際、宣言済みのシグネチャに基づいて型検査し、
  型が合わない節はロード時にエラーとして弾く。
- **`tp/3`**: 値が指定した型(カインド)に適合するかどうかを判定する、
  サブタイピングを考慮した中心的な型判定述語。
- **`wf_kind/1`**: ある項が(いずれかの)カインドとして正しく整形されているか
  を判定する。
